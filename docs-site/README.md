# Trading Research Assistant - Documentation Site

Production-ready documentation site built with React, Vite, and TailwindCSS.

## Important: Source of Truth

**The source of truth for all documentation content is `../docs/` (relative to this folder).**

- Documentation markdown files must be placed in `../docs/<lang>/` (e.g., `../docs/en/`, `../docs/ru/`)
- The site automatically detects languages from folder names in `../docs/`
- Do NOT duplicate markdown files inside `docs-site/src` - all content comes from `../docs/`
- The `prepare-docs.js` script copies files from `../docs/` to `public/docs/` (dev) or `dist/docs/` (build)

## Adding Documentation

1. Add markdown files to `../docs/<lang>/` (e.g., `../docs/en/`, `../docs/ru/`)
2. Languages are auto-detected from folder names in `../docs/`
3. Any new language folder added to `../docs/` will automatically appear in the UI
4. Optional: Add `../docs/<lang>/_nav.json` for custom navigation order and grouping

## Manifest Format

The site uses a generated manifest (`docs-manifest.json`) that contains:

- `languages`: Array of detected language codes (e.g., `["en", "ru"]`)
- `filesByLanguage`: Object mapping language codes to arrays of file paths relative to `docs/` (e.g., `{"en": ["en/overview.md", "en/architecture.md"]}`)
- `generated`: ISO timestamp of when the manifest was generated

The manifest is automatically generated by `prepare-docs.js` and should not be edited manually.

## Deployment

Build outputs to `dist/` folder, ready for GitHub Pages or any static hosting. The site uses HashRouter for compatibility with GitHub Pages.
